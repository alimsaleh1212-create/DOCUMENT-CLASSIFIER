# MinIO Bucket Layout

## Buckets

### `documents`
- Keys: `documents/{batch_id}/{document_id}.tif`
- Source: SFTP ingest worker uploads here
- Access: API generates presigned GET URLs for overlay display

### `overlays`
- Keys: `overlays/{batch_id}/{document_id}.png`
- Source: Inference worker renders and uploads here
- Access: API generates presigned GET URLs for frontend display

## Presigned URLs

- TTL: **900 seconds** (15 minutes)
- Generated by: `app/infra/blob.py` → `MinioBlob.presigned_get(bucket, key, ttl=900)`
- Returned in: `PredictionOut.overlay_url`

## Lifecycle

1. SFTP ingest worker uploads original TIFF to `documents/{batch_id}/{document_id}.tif`
2. Inference worker reads TIFF from `documents/`, classifies, renders overlay PNG
3. Inference worker uploads overlay to `overlays/{batch_id}/{document_id}.png`
4. API generates presigned URL for overlay when serving `GET /batches/{bid}`
5. Both buckets are created at startup if missing
